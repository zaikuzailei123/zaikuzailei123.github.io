引言

阅读GAT的实现时候，发现对稀疏矩阵的处理读不懂，然后就来学习一下稀疏矩阵的常用方式。

COO（Coordinate）

这是最简单的一种格式，每一个元素需要用一个三元组来表示，分别是 （行号, 列号, 数值） ，对应上图右边的一列。这种方式简单，但是记录单信息多（行列），每个三元组自己可以定位，因此空间不是最优的。

class scipy.sparse.coo_matrix(arg1, shape=None, copy=False)

# 四种被实例化的方法：
coo_matrix(D):  # with a dense matrix D  

coo_matrix(S):  # with another sparse matrix S(equivalent to S.tocoo())  

coo_matrix((M,N), [dtype]) # 创建一个M行N列空的稀疏矩阵

# 创建一个矩阵，其中i[:]表示行，j[:]表示列，data表示数据，三个向量每一列表示一个数据，也就是A[i[k],j[k]] = data[k]，A是创建后的稀疏矩阵
coo_matrix((data, (i,j)), [shape=(M,N)])

这种矩阵支持加、减、乘、除、幂运算。
优点：

  • 能够快速建立稀疏矩阵
  • 能够快速转换成其他形式稀疏矩阵
  • 允许重复输入

缺点：

  • 不直接支持数学运算
  • 矩阵切片困难

CSR（Compressed Sparse Row）

CSR需要三类数据来表达：数值，列号，行偏移。CSR不是三元组而是整体的编码方式。数值与列号和COO一致，表示一个元素及其列号，行偏移表示某一行的第一个元素在data里面的起始偏移位置.

![id2][id2]

这样的稀疏矩阵，rows指明行索引，column指明列号，这样就可以快速将一个元素进行定位。例如如果要访问A[i][j], 那么首先根据行号i定位到其所在行第一个元素在data中的位置，然后在data中的该处顺序查找column，当column号和j相等时，输出data。

注意 采用这种方法存放data数据必须先行后列存储，这样定位到row后才能够继续顺序访问查找column

csr_matrix(D)： # 用稠密矩阵D创建稀疏矩阵
  

csr_matrix(S): # 用稀疏矩阵S创建，等同于S.tocsr()


csr_matrix((M,N),[dtype]): # 创建一个空的稀疏矩阵  

csr_matrix((data, (row_ind, col_ind)), [shape=(M,N)]):  # 用COO稀疏矩阵创建一个CSR矩阵


csr_matrix((data, indices, indptr), [shape=(M,N)])      # csr_matrix的标准方法，其中indices保存列号，indptr保存每一行在数据集中的偏置，其中第i行元素的列号保存在indices[indptr[i]:indices[i+1]]，（切片:是前毕后开的），并且他们的数据存放在data[indptr[i]:indptr[i+1]]中。

优势：

  • 对于CSR矩阵之间加法、乘法很高效
  • 行切片很高效
  • 矩阵矢量积很快

劣势：

  • 列切片很慢
  • 稀疏矩阵的结构更改开销大（如增加一个非零元素）

CSC

与CSR类似，只是是对列计算偏置。需要注意的是，底层数据必须按先列后行存储。

优势：

  • CSC矩阵之间做加法乘法开销小
  • 列切片开销小
  • 快速矩阵向量相乘

劣势：

  • 行切片慢
  • 结构改变开销较大（考虑LIL或者DOK）

关键字词典（DOK）

DOK由一个将（行、列）作为关键字映射到元素值的字典组成。字典中缺少的元素将被视为零。这种格式有利于按随机顺序递增地构造稀疏矩阵，但不利于按字典顺序迭代非零值。通常用这种格式构造一个矩阵，然后转换成另一种更有效的格式进行处理。

优势：

  • 支持加减乘除幂运算
  • O（1）时间访问元素
  • 可以轻易地构建coo_matirx矩阵

嵌套列表（LIL）

LIL每行存储一个列表，每个条目包含列索引和值。通常，这些条目按列索引进行排序，以便更快地查找。这是另一种适合增量矩阵构造的格式。
